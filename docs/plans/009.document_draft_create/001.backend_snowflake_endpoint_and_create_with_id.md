# Step 001 — Backend: snowflake endpoint and document create with client-supplied id

## Goal

Expose a new admin endpoint that returns a freshly-generated snowflake id, and
extend `create_document` so it accepts (and validates) an optional
client-supplied id, returning HTTP 409 on collision.

## Files to create or modify

- `backend/app/routes/admin/ids.py` — new thin route module hosting
  `GET /api/admin/snowflake/new`. (If the coder finds a more natural existing
  module under `backend/app/routes/admin/` to extend instead, that is
  acceptable as long as the URL path stays exactly `/api/admin/snowflake/new`
  and the module is registered in the admin router include.)
- `backend/app/routes/admin/__init__.py` — register the new router.
- `backend/app/models/schemas/worlds.py` — extend `CreateDocumentRequest`
  with optional `id: str | None = None`.
- `backend/app/services/world_editor.py` — `create_document`: if `req.id` is
  present, parse it to int and use it as the new row's id; on collision
  raise an HTTP 409.
- `backend/app/routes/admin/worlds.py` — `create_document` route: surface the
  service's 409 (HTTPException) cleanly. No business logic in the route.
- `backend/tests/...` — new test module(s) for the id endpoint and the
  create-with-id behavior. Path follows the existing admin test layout
  (coder picks the matching folder, e.g.
  `backend/tests/routes/admin/test_ids.py` and
  `backend/tests/services/test_world_editor_create_document_with_id.py`).
- `backend/CLAUDE.md` — only if the folder shape changes (new module added).

## Signatures

New schema field:

```
class CreateDocumentRequest(BaseModel):
    id: str | None = None      # NEW — optional, client-supplied snowflake id
    doc_type: ...
    name: ...
    content: ...
    exits: ...
```

New response model (Pydantic) in the same module that hosts the new route:

```
class NewSnowflakeIdResponse(BaseModel):
    id: str
```

New route:

```
@router.get("/snowflake/new", response_model=NewSnowflakeIdResponse)
async def get_new_snowflake_id(...): ...
```

The route calls `app.services.snowflake.generate_id()` directly — pure helper,
no DB. Auth: same admin auth dependency used by sibling admin routes.

Service signature **unchanged externally** (still
`create_document(world_id, req: CreateDocumentRequest) -> DocumentItem` or
whatever the current return type is); internally it now branches on
`req.id is not None`.

## Tests

- `test_ids.py`:
  - `GET /api/admin/snowflake/new` returns 200 with `{ id: "<numeric string>" }`
    when authenticated as admin.
  - Returns 401/403 when unauthenticated (mirror existing admin route tests).
  - Two consecutive calls return distinct ids.
- `test_world_editor_create_document_with_id.py`:
  - Calling `create_document` with `req.id = None` behaves exactly as today
    (regression).
  - Calling `create_document` with `req.id = "<unused snowflake>"` creates the
    row with that exact id.
  - Calling `create_document` with `req.id` matching an existing
    `WorldLocation`, `WorldNPC`, or `WorldLoreFact` id raises HTTP 409 (test
    each of the three tables).
  - Invalid id strings (non-numeric) raise HTTP 422 (Pydantic) or 400 (service);
    coder picks which layer rejects, but the contract is "client gets a 4xx,
    not a 500".

## Definition of done

- `GET /api/admin/snowflake/new` exists, is registered under the admin router,
  is admin-auth-protected, and returns a `{ id: string }` JSON body.
- `CreateDocumentRequest` has an optional `id: str | None = None`.
- `create_document` service uses `req.id` when supplied (parsing to int) and
  falls back to `generate_id()` otherwise.
- Collisions across all three document tables (`WorldLocation`, `WorldNPC`,
  `WorldLoreFact`) produce HTTP 409.
- All three id-assignment call sites in `world_editor.create_document` (lines
  ~612, 677, 762) honor the supplied id.
- No DB queries in the route layer; collision check lives in `db/` or
  `services/` per layer-separation rules.
- No `session` / `AsyncSession` imports outside `backend/app/db/`.
- Tests above all pass.
- No JSONL import/export changes (verified — model shape unchanged).
- `backend/CLAUDE.md` updated only if a new route module was added.
