## Pattern to mirror (do not copy verbatim)

`frontend/src/admin/pages/PipelineEditPage.tsx` and
`pipelineEditPageState.ts:23,112-156` use a `'shadow' | 'edit'` mode for
draft pipelines on a synthetic `/new` URL. We are deliberately **not**
adopting the synthetic URL ‚Äî we have a real pre-allocated id ‚Äî but the
"two-mode state, branch on Save" structure is a useful reference for naming
and shape.

## Empty-draft shape

For each `doc_type` the empty draft should at minimum populate:

- `id`: the pre-allocated snowflake id (from URL).
- `doc_type`: from `?doc_type=...`.
- `name`: empty string.
- `content`: empty string.
- `exits`: undefined / empty list (only relevant for locations).

The exact `DocumentItem` shape is in `frontend/src/types/world.d.ts:62-90` ‚Äî
the coder fills any other required fields with the obvious empty equivalents.

## Save path

`worldsApi.createDocument(worldId, req, signal)` already exists. The new
`req.id` field is the only change at the call site. After success, replay
`pendingLinkOps` sequentially via the existing link APIs; if any replay
fails, surface the error via the existing `saveStatus` async-trio without
rolling back the just-created document (acceptable per the chosen design ‚Äî
the document exists, links can be retried by the user).
