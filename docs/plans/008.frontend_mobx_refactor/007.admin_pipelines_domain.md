# Feature 008 Step 007 — Admin Pipelines Domain

Goal: migrate the three pipeline pages plus the shared `LlmChatPanel` to the canonical pattern. Restore cursor-position placeholder insertion (deferred from step 003b).

## Pages and components migrated

| Path / File | Approx LOC | Shape |

## Locked patterns

- Path-param wrappers in `admin/routes.tsx` extract via `useParams` and pass props (composite `key` for stage edit). `/admin/pipelines/new` keeps its literal `key="new"` and the same component, but the page reads `?cloneFrom=` from query params.
- Each page sibling: `<page>PageState.ts` exporting class + external `(state, args, signal)` functions.
- All migrated files observer-wrapped; single mount/unmount `useEffect` per page.
- Same-SPA navigation via `useNavigate`. Same-SPA path crossings (`new`, `:id`) trigger remount via the route shape.
- `LlmChatPanel` keeps its **public props identical** (`currentContent`, `worldId?`, `docId?`, `docType?`, `fieldType?`, `onApply`, `onAppend`). All internal state moves into a sibling state class. Step 006 consumers (`DocumentEditPage`, `WorldFieldEditPage`) continue to work without page-side changes.

## Per-page shape

### `PipelinesListPage` + `pipelinesListPageState.ts`

- Fields: `pipelines` trio.
- External: `loadPipelines(state, signal)`, plus `deletePipeline(state, pipelineId, signal)` if the page exposes delete (per current behavior).
- Actions:
  - Row click → `navigate('/pipelines/' + p.id)` (router basename adds `/admin`).
  - Clone → `navigate('/pipelines/new?cloneFrom=' + p.id)`.
  - Create new → `navigate('/pipelines/new')`.

### `PipelineEditPage` + `pipelineEditPageState.ts`

Constructor: `pipelineId: string | null` (null when on `/new`), `cloneFromId: string | null` (from `?cloneFrom=`).

- Fields:
  - `mode: 'shadow' | 'edit'` — derived from constructor (shadow when `pipelineId === null`).
  - `pipeline: PipelineItem | null` + trio (only meaningful in edit mode; in shadow mode `loadStatus` skips to `'ready'` after pulling clone source if present).
  - `configOptions: PipelineConfigOptions | null` + trio.
  - `draft: PipelineDraft` (name, description, kind, system_prompt, simple_tools, pipeline_config — JSON of stages, agent_config). Initialized from `pipeline` (edit) / clone source / blank (shadow).
  - `serverErrors`, `saveStatus`, `saveError`.
- Computed: `errors`, `isDirty`, `isValid`, `canSubmit`. In shadow mode `isDirty` is true if any field differs from blank/clone-source.
- External:
  - `loadPipelineEdit(state, signal)` — concurrent: load `pipeline` (edit) or load `cloneFromId` source if present (shadow); load `configOptions`.
  - `savePipeline(state, signal): Promise<string | null>` — POST in shadow mode, PUT in edit mode. Returns new id on shadow create (page navigates).
  - `addStage(state, stageType)`, `removeStage(state, idx)`, `reorderStages(state, fromIdx, toIdx)` — local mutations on `draft.pipeline_config.stages`. Drag-reorder UI calls `reorderStages`.
- Cross-page navigation:
  - Stage row → `navigate('/pipelines/' + id + '/stage/' + idx)` (only in edit mode — shadow stages aren't persisted yet; either disable navigation or save-on-click — keep current behavior).
  - Back → `navigate('/pipelines')`.
  - Clone-this → `navigate('/pipelines/new?cloneFrom=' + id)`.
- Splitting: extract `PipelineGeneralForm`, `PipelineStagesList` as observer subcomponents under `admin/components/pipelines/` if it improves readability (encouraged but not mandatory). State stays on the page.

### `PipelineStageEditPage` + `pipelineStageEditPageState.ts`

Constructor: `pipelineId: string`, `stageIndex: number`.

- Fields:
  - `pipeline: PipelineItem | null` + trio.
  - `configOptions: PipelineConfigOptions | null` + trio.
  - `enabledModels: EnabledModelInfo[]` + trio.
  - Stage draft: `content: string` (prompt), `stageEnabled: boolean`, `stageModelId: string | null`, plus the read-only fields surfaced by the page (`stageName`, `stepType`, `stageTools`).
  - `originalContent: string` for `isDirty`.
  - `saveStatus`, `saveError`, plus `successMessage` (transient — current page shows "Applied." for 3s).
- Computed: `isDirty` (compares content/stageEnabled/stageModelId against original), `canSubmit`.
- External: `loadStage(state, signal)`, `saveStage(state, signal)` — applies the change to the pipeline's `pipeline_config.stages[stageIndex]` via `updatePipeline`. `loadDefaultTemplate(state)` — local mutation that replaces content with the appropriate default template (with confirm).
- **Restore cursor-position placeholder insert** (deferred from step 003b): expose an imperative ref out of `PlaceholderTextarea` so the badge-click handler in `PipelineStageEditPage` can read the current caret and insert at cursor. Two clean options:
  - **Option A**: `PlaceholderTextarea` accepts an optional `controllerRef?: RefObject<{ insertAtCursor: (text: string) => void }>` prop. The textarea exposes a stable controller that the page calls.
  - **Option B**: `PlaceholderTextarea` accepts an optional `imperativeRef?: RefObject<HTMLTextAreaElement | null>` prop and assigns its internal `textareaRef` into it on mount; the page reads `current.selectionStart` directly.
  - Pick **Option A** — keeps the textarea's DOM ref private and exposes a small, named imperative API.

### `LlmChatPanel` + `llmChatPanelState.ts`

`admin/components/llm/LlmChatPanel.tsx` keeps its public props. Internal state moves to a sibling `llmChatPanelState.ts` class instance owned via `useState(() => new LlmChatPanelState())`.

- Fields:
  - Models: `models: EnabledModelInfo[]` + trio; `selectedModel: string | null` (localStorage-backed).
  - Params: `params: EditorLlmParams` (localStorage-backed); `paramsOpen: boolean`.
  - Messages: `messages: ChatMessage[]`.
  - Streaming: `isStreaming`, plus a non-observable `abortCtrl: AbortController | null`.
- Computed: `modelOptions` (Mantine `Select` data shape derived from `models`).
- External:
  - `loadModels(state, signal)` — fetches enabled models; if saved `selectedModel` isn't in the list, clear it.
  - `sendChatMessage(state, props)` — appends user message, kicks off `streamChat`, accumulates token/thinking/tool callbacks.
  - `stopChat(state)` — aborts.
  - `regenerateLast(state, props)` — drops last assistant message, re-streams.
  - `regenerateAtMessage(state, messageId, props)` — truncates after the message, re-streams.
  - `deleteMessage(state, messageId)`, `clearMessages(state)`.
  - `setSelectedModel(state, modelId)` (mutator + localStorage write).
  - `setParam(state, key, value)` (mutator + localStorage write).
- The `props` argument to send/regenerate carries the request context (`currentContent`, `worldId`, `docId`, `docType`, `fieldType`) the panel needs from its caller.
- `MessageBubble` and `ToolCallRow` stay as inline observer subcomponents. Each takes the message (or call) plus the `onApply` / `onAppend` / `onDelete` / `onRegenerate` callbacks.
- Auto-scroll: keep current behavior. The "scroll-to-bottom when at bottom" effect is the rare allowed `useEffect` — but it's tied to the messages-changed signal which is now observable. Refactor to a MobX reaction (`autorun` set up in the panel's mount-effect, disposed on cleanup) **OR** keep a single `useEffect([state.messages.length])` with the messages-length dep. Pick whichever is cleaner; the autorun approach matches the architecture better.

PipelineEditPage routes:
  '/pipelines/new'           → <PipelineEditPage key="new" pipelineId={null} cloneFromId={searchParams.get('cloneFrom')} />
