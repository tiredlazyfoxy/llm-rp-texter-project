# Feature 008 Step 003 — Component Folder Reorg

Goal: organize per-SPA component folders into domain subfolders matching `frontend-layout.md`. Pure file moves + import-path updates. No internals modified, no behavior change.

This step prepares the ground for step 003b (which adds new wrapper components) and for later page-migration steps that may further split components.

## Decisions

- **Cross-SPA shared components stay flat at `src/components/`**: `AppLayout`, `AppHeader`, `AppSidebar`, `ChangePasswordModal`, `TranslationSettingsModal`. No moves.
- **`UserSidebar.tsx` stays at top-level `user/components/`** as a layout-shell exception (not a domain component). Document the exception in CLAUDE.md.
- **`common/` subfolders are NOT created in this step.** They appear in step 003b alongside the new wrapper components.
- **`LlmChatPanel` lands in `admin/components/llm/`** — it's an LLM-chat tool used across editing surfaces; not generic enough for `common/`.
- **`PlaceholderPanel` and `PlaceholderSuggestions` go to `admin/components/pipelines/`** — pipeline-domain UI.

## Moves

**User SPA** (`user/components/<name>.tsx` → new path):

| File | Destination |
|------|-------------|
| `ChatInput.tsx`, `ChatMemoriesModal.tsx`, `ChatSettingsPanel.tsx`, `MessageBubble.tsx`, `MessageHistory.tsx`, `StatsPanel.tsx`, `SummaryBlock.tsx`, `ToolCallTrace.tsx` | `user/components/chats/` |
| `WorldInfoModal.tsx` | `user/components/worlds/` |
| `UserSidebar.tsx` | unchanged |

**Admin SPA**:

| File | Destination |
|------|-------------|
| `CreateUserModal.tsx`, `SetPasswordModal.tsx`, `SetRoleModal.tsx` | `admin/components/users/` |
| `PlaceholderPanel.tsx`, `PlaceholderSuggestions.tsx` | `admin/components/pipelines/` |
| `LlmChatPanel.tsx` | `admin/components/llm/` |

After the moves, `admin/components/` top level contains only the three subfolders (no loose files).

## Import updates

Find every caller of a moved file and update its import path. `tsc --noEmit` catches misses.

Two classes of imports to update:

- External (page or another component imports a moved component) — fix the path.
- Internal (a moved component imports a sibling that also moved) — usually unchanged (same `./`); only changes when a moved file imports a NOT-moved sibling like `UserSidebar`.

## CLAUDE.md updates

- `frontend/src/user/CLAUDE.md` — refresh layout block to reflect `chats/`, `worlds/`, top-level `UserSidebar.tsx` exception. Note `routes.tsx` (added in step 001).
- `frontend/src/admin/CLAUDE.md` — refresh layout block to reflect `users/`, `pipelines/`, `llm/` subfolders. `hooks/` stays for now (deleted in step 003b).

## What does NOT change

- Component internals — every moved file is byte-identical at its destination apart from possible same-folder import path tweaks.
- Pages, routes, api/, types/, utils/, auth.ts, the two custom hooks.
- Cross-SPA shared `src/components/`.
- No `observer` wrapping, no MobX additions, no new components.

## Verification

- `npm run build` succeeds.
- Grep `frontend/src` for any stale path import (e.g., `from "../components/ChatInput"`, `from "../components/LlmChatPanel"`, etc.) — must return zero matches.
- Folder shape matches the move table.
- Dev-server smoke: log in, exercise each domain (user chat, world info modal, admin worlds/pipelines/users editing screens, AI-edit panels). No module-not-found errors.

## Acceptance Criteria

1. Folder shape matches the move table; `admin/components/` top level has no loose files.
2. `frontend/src/components/` (cross-SPA) is unchanged.
3. `tsc` passes with zero errors.
4. Grep for old paths returns zero matches.
5. Dev-server smoke confirms parity with pre-step behavior across all migrated component domains.
6. CLAUDE.md files reflect the new layout.
7. Custom hooks (`hooks/useTranslation.ts`, `admin/hooks/usePlaceholderAutocomplete.ts`) still exist and are unchanged.

## Out of scope

- Creating `common/` subfolders — step 003b.
- Replacing the custom hooks — step 003b.
- Wrapping components in `observer` — happens during their page's migration step.
- Splitting monolithic pages into domain subcomponents — page-migration steps.
- Moving `UserSidebar.tsx` to a `layout/` subfolder — not worth the import churn for a single file; revisit if a second SPA-layout shell appears.
