## Audits to run

Each audit is a grep over `frontend/src` (excluding `node_modules`, `dist`). Any non-zero hit is either fixed in this step or explicitly justified.

### A1. `useState` for reactive state

- Pattern: `useState\(` not immediately followed by `\(\) => new `.
- Allowed: `useState(() => new XPageState(...))`, `useState(() => new XDraft())`, `useState(() => new XControllerState())` — all instance-memoization patterns.
- Disallowed: `useState("")`, `useState(false)`, `useState<T>(initial)`, etc., for reactive state.
- Action: convert to a MobX observable on the appropriate state class. If a value is genuinely non-reactive (e.g., a stable controller from a third-party API), document why with a brief inline comment.

### A2. `useEffect` discipline

- Allowed pattern: exactly one `useEffect(() => { ...; return () => ...; }, [])` per page (mount-load + unmount-cleanup).
- Disallowed: `useEffect` in leaf components, `useEffect` watching props/observables, `useEffect` with non-empty deps for derivation.
- Exception: a single `useEffect([opened])` reset on a modal where the component is mounted continuously (e.g., it's part of a kept-mounted parent). If unavoidable, document the exception inline.
- Action: replace derivation effects with `get` computeds; replace prop-watching effects with router `key` remount; move loaders to mount/unmount pairs.

### A3. `useCallback` / `useMemo`

- Disallowed everywhere. Observer rerenders are scoped; ref stability is irrelevant.
- Action: delete and inline.

### A4. `useReducer`

- Disallowed.
- Action: replace with MobX observable + actions.

### A5. Custom `useX` hooks

- No custom hooks under `frontend/src` matching `^export function use[A-Z]` or `^function use[A-Z]`.
- `frontend/src/hooks/` and `frontend/src/admin/hooks/` directories must not exist.
- Action: convert to a wrapper component owning a `<Component>State` class (per step 003b pattern).

### A6. Observer wrapping

- Every component file must export at least one `observer(...)` (or use `observer` as the default for the wrapping pattern in use).
- Pattern: every `export const X = (...) => { ... }` or `export function X(...) { ... }` for a React component must go through `observer`.
- Allowed exception: pure presentational components that only take primitives + callbacks and have no chance of reading observables today. List them explicitly inline if any. Default policy is "wrap everything." Missing `observer` is a silent staleness risk.
- Action: wrap with `observer`.

### A7. Effectful methods on state classes

- No method on a `<Page>State` / `<Component>State` class that calls `api/...`, opens an SSE stream, or has external side effects.
- Allowed: `dispose()` (aborts the state's controller), `toJSON()`-like helpers, pure derivations as `get`.
- Action: extract to an external `(state, args, signal)` function in the same file.

### A8. Direct `fetch` outside `api/`

- `fetch(` calls allowed only inside `frontend/src/api/`. State files, pages, and components must use the resource modules.
- Action: move to the appropriate `api/<resource>.ts` and call from there.

### A9. `window.location` reads / writes

- Allowed: `window.location.href = "/login/"` and similar **cross-SPA** redirects (login entry, logout from auth.ts, "Admin" link from user header, "Logo" cross-SPA link). These are not router-routable.
- Disallowed elsewhere: `window.location.pathname`, `window.location.search`, `window.location.href = <same-SPA-url>`. Replace with `useNavigate`, `useParams`, `useSearchParams`.
- Action: convert.

### A10. React Context

- No `createContext` / `useContext` anywhere.
- Action: convert to prop drilling. (Cross-SPA shared shells are passed through props, not context.)

### A11. Singletons / module-level mutable state

- Allowed module-level state: `auth.ts` (token + user via `localStorage`), `utils/translationSettings.ts` (settings cache + load/save), and similar global-settings caches with the same get/save shape.
- Disallowed: any singleton MobX store, any `export const xStore = new XStore()` pattern.
- Action: confirm none exist after step 005.

### A12. `models/` folder

- `frontend/src/models/` must not exist; types live in `types/`.

### A13. `appState.ts` / root MobX store

- Must not exist.

### A14. `from "../api/request"` (legacy)

- Must return zero matches (the file was deleted in step 002).

### A15. `chatStore` symbol

- Must return zero matches (deleted in step 005).

### A16. `useTranslation` and `usePlaceholderAutocomplete` symbols

- Must return zero matches (deleted in step 003b).

### A17. Async resource trio shape

- Every loadable on a state class follows `<name>` + `<name>Status` + `<name>Error` (`AsyncStatus = 'idle' | 'loading' | 'ready' | 'error'`).
- Disallowed: `isLoading: boolean`, `loading: boolean`, `pending: boolean` for resource-load state.
- Action: convert.
