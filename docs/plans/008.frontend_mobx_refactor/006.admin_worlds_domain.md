# Feature 008 Step 006 — Admin Worlds Domain

Goal: migrate the admin worlds + documents pages to the canonical `<Page>State` pattern. Five pages, ~2000 lines total — the largest admin cluster.

## Pages migrated

| Path | Page | Approx LOC | Shape |
|------|------|------------|-------|
| `/admin/worlds` | `WorldsListPage` | 196 | List + inline `CreateWorldModal` |
| `/admin/worlds/:worldId` | `WorldViewPage` | 575 | Tabbed read-only view (Info / All Docs / Locations / NPCs / Lore Facts / Chats) |
| `/admin/worlds/:worldId/edit` | `WorldEditPage` | 624 | Edit form + clone + cross-page links to field/document edit |
| `/admin/worlds/:worldId/field/:fieldName` | `WorldFieldEditPage` | 224 | AI-assisted field editor (description / initial_message) — uses `LlmChatPanel` |
| `/admin/worlds/:worldId/documents/:docId/edit` | `DocumentEditPage` | 337 | Document editor — uses `LlmChatPanel` |

## Locked patterns

- Path param flows from the route wrapper. Wrappers in `admin/routes.tsx` extract `useParams()` and pass `worldId` (and `docId` / `fieldName` where applicable) as both `key` (composite when multiple params) and props.
- Each page has a sibling `<page>PageState.ts` exporting the class + external `(state, args, signal)` functions.
- Every page and any subcomponent that reads MobX state is `observer`-wrapped.
- Single mount/unmount `useEffect` per page; cleanup aborts the load `AbortController`.
- Same-SPA navigation via `useNavigate`; cross-SPA (none in this domain) — N/A.
- Async resource trio for every loadable: `<name>` / `<name>Status` / `<name>Error`.
- **Form pages** use the draft pattern from `frontend-forms.md`: `world: World | null` (server snapshot), `draft: WorldDraft` (editable copy), `serverErrors: Partial<DraftErrors>`, `saveStatus`, `saveError`. Validation as `get` computed; `get errors`, `get isDirty`, `get isValid`, `get canSubmit`. Submit = inner closure → guard `canSubmit` → external `saveX(state, signal)`.
- **Modal create dialogs** use component-local draft (small class via `useState(() => new XDraft())`), per architecture.
- **Splitting** (`frontend-components.md`): `WorldViewPage` and `WorldEditPage` are large enough that splitting into observer subcomponents per tab/section is encouraged. Subcomponents take `state` (or a slice) by prop. State stays on the page.

## Per-page shape

### `WorldsListPage` + `worldsListPageState.ts`

- Fields: `worlds` trio.
- External: `loadWorlds(state, signal)`.
- `CreateWorldModal` stays inline as a top-level subcomponent in the page file. Draft uses local `useState(() => new CreateWorldDraft())`. Draft fields: `name`, `description`, `status`, `serverErrors`. Computed: `errors`, `isValid`, `canSubmit`. External `createNewWorld(draft, signal): Promise<WorldItem | null>` next to the draft class. On success, closes modal and refreshes the list (parent passes a `onCreated` callback that calls `loadWorlds(state, signal)` again).

### `WorldViewPage` + `worldViewPageState.ts`

Constructor: `worldId`. Reads tab from URL query (`?tab=...`), mirrors back on change.

- Fields:
  - `world: WorldDetail | null` + trio.
  - `documents`, `locations`, `npcs`, `loreFacts`, `chats` — each its own trio (lazy-loaded per-tab or eager — keep current behavior; today the page loads via `getWorld(worldId)` which returns the bulk plus uses `listDocuments` for filtering).
  - `tab: WorldViewTab` (URL-mirrored; default `'info'`).
- External: `loadWorld(state, signal)` (populates the world + nested lists from the existing endpoints), plus per-tab refresh helpers as needed. Action externals: `deleteDocument`, `reindexWorld`, etc., per current behavior.
- Splitting suggestion: extract `WorldInfoTab`, `WorldDocsTab`, `WorldNpcsTab`, etc. as observer subcomponents under `admin/components/worlds/` (creates that folder). Each takes the page state as prop.

### `WorldEditPage` + `worldEditPageState.ts`

Constructor: `worldId`. Form page.

- Fields:
  - `world: WorldDetail | null` + trio.
  - `pipelineOptions` (pipeline picker dropdown data) + trio.
  - `draft: WorldDraft` — fields mirror `UpdateWorldRequest` (`name`, `description`, `pipeline_id`, `status`, etc., per current page). Initialized from `world` after load.
  - `serverErrors: Partial<WorldDraftErrors>`.
  - `saveStatus`, `saveError`.
- Computed: `errors`, `isDirty` (shallow compare draft vs world), `isValid`, `canSubmit`.
- External: `loadWorldEdit(state, signal)` (loads world + pipelines concurrently), `saveWorld(state, signal)`, `cloneWorld(state, signal): Promise<string | null>` (returns new world id; page navigates), `deleteWorld(state, signal)`.
- Cross-page navigation handled via `useNavigate`:
  - Field-edit AI button → `/admin/worlds/<id>/field/<name>`.
  - Document-edit row click → `/admin/worlds/<id>/documents/<docId>/edit`.
  - Back → `/admin/worlds/<id>`.
  - Clone success → `/admin/worlds/<newId>`.
- Splitting suggestion: extract `WorldEditForm` (or `WorldGeneralSection`), `WorldDocsSection`, `WorldStatsSection`, `WorldRulesSection`, `WorldNpcLinksSection` as observer subcomponents under `admin/components/worlds/`. Each gets page-state by prop.

### `WorldFieldEditPage` + `worldFieldEditPageState.ts`

Constructor: `worldId`, `fieldName`. AI-assisted field editor.

- Fields:
  - `world: WorldDetail | null` + trio (only the field's current value is needed, but the load mirrors current behavior).
  - `draft: string` (the field content).
  - `originalContent: string` for `isDirty`.
  - `saveStatus`, `saveError`.
- Computed: `isDirty`, `canSubmit`.
- External: `loadField(state, signal)`, `saveField(state, signal)` — calls `updateWorld` with `{ [fieldName]: draft }` semantics.
- Renders `LlmChatPanel` for AI assist; the panel's `onApply` / `onAppend` callbacks mutate `state.draft`.

### `DocumentEditPage` + `documentEditPageState.ts`

Constructor: `worldId`, `docId`. Document editor.

- Fields:
  - `doc: DocumentItem | null` + trio.
  - `draft: DocumentDraft` (title + content + metadata fields per current shape).
  - `serverErrors`.
  - `saveStatus`, `saveError`.
- Computed: `errors`, `isDirty`, `isValid`, `canSubmit`.
- External: `loadDocument(state, signal)`, `saveDocument(state, signal)`, `deleteDocument(state, signal)`.
- Renders `LlmChatPanel` with doc context (`docId`, `docType`) for AI assist.

## Acceptance Criteria

1. Five new state files exist and export their class plus the listed external functions.
2. The five pages are observer-wrapped, use `useState(() => new ...PageState(...))`, and have exactly one mount/unmount `useEffect`.
3. `WorldViewPage`, `WorldEditPage`, `WorldFieldEditPage`, `DocumentEditPage` accept their respective path-param props; `WorldsListPage` accepts no props.
4. `admin/routes.tsx` wrappers extract path params via `useParams` and pass them as both composite `key` and props.
5. No `window.location.pathname` reads in the migrated files.
6. Same-SPA navigation in the migrated pages uses `useNavigate`.
7. Form pages have `draft` / `serverErrors` / `saveStatus` separation, `get errors` / `get canSubmit` computeds; submit guarded by `canSubmit`; external `saveX(state, signal)` does the API call and translates 422s into `serverErrors`.
8. `WorldsListPage`'s `CreateWorldModal` uses a component-local draft class (small `useState(() => new ...)` pattern) — not page state.
9. `LlmChatPanel` consumers (`WorldFieldEditPage`, `DocumentEditPage`) integrate via the panel's existing props (`onApply`, `onAppend`) — page state mutated by the callbacks.
10. `tsc` passes with zero errors.
11. Smoke-test paths above all behave at parity with the pre-step pages.
12. `admin/CLAUDE.md` reflects the new layout. If `admin/components/worlds/` is created, it's listed.
