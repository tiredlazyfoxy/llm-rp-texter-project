# Feature 008 Step 003b — Hooks → Wrapper Components

Goal: replace the two custom hooks (`useTranslation`, `usePlaceholderAutocomplete`) with wrapper components owning internal `<Component>State` MobX classes. Migrate the three call sites. Move `TranslateStreamHandlers` and `TranslateStreamFn` types to `types/llmChat.d.ts`. Delete `frontend/src/hooks/` and `frontend/src/admin/hooks/`.

This step makes the architecture rule "no custom `useX` hooks; reusable stateful UI behavior is a wrapper component owning a state class" enforceable by grep.

## Locked design

- **Both new components are CONTROLLED inputs**: caller passes `value` + `onChange`, the component owns side-state internally. This avoids forcing observer/MobX onto callers that aren't migrated yet (LlmChatPanel, PipelineStageEditPage).
- **`LlmInputBar`** = variant C with controlled value. Built-in: textarea, translate, revert, error banner, send/stop swap. Slot props: `before`, `extras`. Internal state class: `LlmInputState`. Lives at `src/components/` (cross-SPA shared, flat).
- **`PlaceholderTextarea`** = controlled textarea + autocomplete dropdown. Internal state class: `PlaceholderAutocompleteState`. Reuses the existing `PlaceholderSuggestions` component as the dropdown. Lives at `admin/components/pipelines/`.
- **`observer`** is NOT added to caller pages (`LlmChatPanel`, `PipelineStageEditPage`). Their migration to MobX is a later step. The new wrapper components are observer-internally; that's enough for their MobX subtree.

## New files

### `frontend/src/components/llmInputState.ts`

Holds:

- Class `LlmInputState` — observable fields covering: `isTranslating`, `canRevert`, `translateError`, plus internal buffers (`originalText`, `translatedText`, mid-stream thinking/token accumulators) and a non-observable `abortCtrl`.
- External functions taking `(state, ...args)`:
  - `startTranslate(state, translateFn, value, setValue)` — reads global translation settings, fails fast with a friendly error if no model is configured, aborts any prior translate, streams thinking → tokens → done into the textarea via `setValue`. On error, restores original text.
  - `revertTranslate(state, setValue)` — restores original; clears revert state.
  - `stopTranslate(state)` — aborts in-flight translate.
  - `clearTranslateError(state)`.
  - `onValueEdit(state, newValue)` — call after every textarea change; if user manually edits while in revertable state, drops the revert buffer.

### `frontend/src/components/LlmInputBar.tsx`

Observer component. Props:

```ts
interface LlmInputBarProps {
  value: string;
  onChange: (v: string) => void;
  translateFn?: TranslateStreamFn;   // optional: omit to hide translate UI
  busy?: boolean;                    // external busy → disables input, shows stop
  onSend: () => void;
  onStop?: () => void;               // when busy, wired to the stop button
  disabled?: boolean;                // hard-disable (e.g., chat archived)
  placeholder?: string;
  before?: ReactNode;                // slot above textarea
  extras?: ReactNode;                // slot after send/stop button
  submitOnEnter?: boolean;           // default true; Shift+Enter = newline
  textareaProps?: Omit<TextareaProps, "value"|"onChange"|"onKeyDown"|"disabled"|"placeholder">;
}
```

Owns its `LlmInputState` via `useState(() => new LlmInputState())`. Renders:

- Optional `before` slot.
- Translate-error banner (clickable to dismiss) when `tx.translateError`.
- A row containing: textarea, translate button (when `translateFn` given), revert button (when `tx.canRevert`), send-or-stop swap, optional `extras`.

Disable rules: input disabled when `disabled || busy || tx.isTranslating`; send disabled when input empty or hard-disabled; stop disabled when no `onStop` provided.

### `frontend/src/admin/components/pipelines/placeholderAutocompleteState.ts`

Holds:

- `interface AutocompletePosition { top: number; left: number; }`
- Class `PlaceholderAutocompleteState` — observable fields: `visible`, `suggestions`, `selectedIndex`, `position`. Plus a non-observable stable `textareaRef: RefObject<HTMLTextAreaElement | null>` created once at construction.
- Pure helper `getCaretPixelPosition(ta, caretPos)` (verbatim port of the existing mirror-div approach from the deleted hook).
- Pure helper `getPartial(text, cursorPos)` — returns the placeholder partial after the last unclosed `{`, or null.
- External functions:
  - `onTextChange(state, placeholders, value, ta)` — update suggestions/visibility/position based on caret and partial.
  - `handleKeyDown(state, event, value, setValue)` — Arrow up/down navigate, Enter/Tab apply, Escape dismiss.
  - `applySelection(state, index, value, setValue)` — replace `{PARTIAL` with `{NAME}` and dismiss; preserve cursor position via `requestAnimationFrame`.
  - `dismiss(state)`.

### `frontend/src/admin/components/pipelines/PlaceholderTextarea.tsx`

Observer component. Props:

```ts
interface PlaceholderTextareaProps {
  value: string;
  onChange: (v: string) => void;
  placeholders: PlaceholderInfo[];
  textareaProps?: Omit<TextareaProps, "value"|"onChange"|"onKeyDown"|"ref">;
}
```

Owns `PlaceholderAutocompleteState` via `useState(...)`. Renders Mantine `<Textarea>` (binding the state's `textareaRef`) plus `<PlaceholderSuggestions>` driven by the autocomplete state.

## Migration of existing callers

### `user/components/chats/ChatInput.tsx`

- Drop the `useTranslation` import + call, the inline translate/revert/stop/send/regenerate buttons, the inline error banner.
- Keep the resize-handle wrapper, OOC preview, status row (loader + phase badge + status text), local `useState(value)`, observer wrapping (already present).
- Render `<LlmInputBar>` with: `value`/`onChange` from local state; `translateFn={translateTextChat}`; `busy={chatStore.isSending}`; `onSend`/`onStop`; `disabled` when chat archived; OOC preview + status row in `before`; the regenerate button (when shown) in `extras`. Pass through Mantine textarea styling via `textareaProps`.

### `admin/components/llm/LlmChatPanel.tsx`

- Drop the `useTranslation` call and the inline translate/revert/send/stop buttons + translate-error JSX.
- Keep model selector, params panel, message list, ToolCallRow/MessageBubble, the regenerate button BELOW the message list (it's outside the input cluster).
- Replace the input cluster with `<LlmInputBar>` using `value`/`onChange` from local state; `translateFn={translateTextAdmin}`; `busy={isStreaming}`; `onStop={handleStop}`; `disabled={!selectedModel}`; autosize textarea props.

### `admin/pages/PipelineStageEditPage.tsx`

- Drop the `usePlaceholderAutocomplete` call, the local `textareaRef`, the inline `<Textarea ref=…>` block, the `<PlaceholderSuggestions>` block.
- Render `<PlaceholderTextarea>` instead, fully controlled via the page's existing `useState(content)`.
- `insertPlaceholder` (the `PlaceholderPanel` badge-click handler) downgrades to **append-at-end** (`setContent(c => c + \`{${name}}\`)`) — current implementation reads `textareaRef.current` for cursor position; that ref now lives inside `PlaceholderTextarea`. Cursor-position insertion can be restored later by forwarding a ref or moving the badge logic inside the textarea component (revisit in step 007).
