# Feature 001 Step 003 — LLM Server Management

## Context

Before the world editor (step 4) can use LLM chat for document editing, we need a management page for LLM servers. Based on the working reference implementation in DatasetControlCenter (`LlmServersPage.tsx` + `llm_servers.py`), this step builds a full server management UI — not a stub.

Editors need to select a model when chatting with the LLM in the document editor. Admins configure which servers exist and which models on each server are approved for use.

---

## 1. LLM Server Model

**File**: `backend/app/models/llm_server.py`

### Table: `llm_servers`

| Field | Type | SQLite Type | Notes |
|---|---|---|---|
| `id` | int64 (snowflake) | INTEGER | Primary key, generated by snowflake generator |
| `name` | str | TEXT | Display name (e.g. "Local llama-swap", "OpenAI Production") |
| `backend_type` | str | TEXT | `"llama-swap"` or `"openai"` (OpenAI-compatible API) |
| `base_url` | str | TEXT | Server URL (e.g. `http://localhost:8080/v1`), trailing slash stripped on save |
| `api_key` | str \| None | TEXT | Optional. Supports `$ENV_VAR_NAME` syntax for env-based resolution |
| `enabled_models` | str | TEXT | JSON array of enabled model ID strings, default `"[]"` |
| `is_active` | bool | INTEGER (0/1) | Whether server is available for use, default `True` |
| `created_at` | datetime | TEXT (ISO) | Auto-set on create |
| `modified_at` | datetime | TEXT (ISO) | Auto-set on create/update |

### SQLModel Definition

```python
class LlmServer(SQLModel, table=True):
    __tablename__ = "llm_servers"

    id: int = Field(primary_key=True)  # snowflake int64
    name: str
    backend_type: str  # "llama-swap" | "openai"
    base_url: str
    api_key: str | None = None
    enabled_models: str = "[]"  # JSON array of model ID strings
    is_active: bool = True
    created_at: datetime
    modified_at: datetime
```

### Key Behaviors

- **ID generation**: Uses the project's snowflake generator (same as all other models)
- **ID serialization**: Stored as INTEGER in SQLite, serialized as **string** in all API responses (JS can't handle int64)
- **API key resolution**: If `api_key` starts with `$`, resolve from environment variable at runtime (never store the resolved value). E.g. `$OPENAI_API_KEY` → reads `os.environ["OPENAI_API_KEY"]`
- **API key security**: Never expose `api_key` in API responses — return `has_api_key: bool` instead
- **`enabled_models` storage**: Stored as JSON string in SQLite, parsed/serialized via `json.loads()`/`json.dumps()` in the API layer
- **`base_url` normalization**: Strip trailing `/` on create/update

## 2. API Endpoints

**File**: `backend/app/routes/llm_servers.py`

All endpoints require **admin** role except the models list endpoint (editor can read).

| Method | Path | Purpose | Role |
|---|---|---|---|
| GET | `/api/admin/llm-servers` | List all servers | admin |
| POST | `/api/admin/llm-servers` | Create server | admin |
| PUT | `/api/admin/llm-servers/:id` | Update server | admin |
| DELETE | `/api/admin/llm-servers/:id` | Delete server | admin |
| GET | `/api/admin/llm-servers/:id/available-models` | Probe server for available models | admin |
| PUT | `/api/admin/llm-servers/:id/enabled-models` | Set which models are enabled | admin |
| GET | `/api/llm/models` | List all enabled models across active servers (for editor model picker) | editor |

### Pydantic Schemas

```
LlmServerResponse: id(str), name, backend_type, base_url, has_api_key(bool), enabled_models(list[str]), is_active, created_at, modified_at
LlmServersListResponse: items(list[LlmServerResponse])
CreateLlmServerRequest: name, backend_type, base_url, api_key?(str|None), is_active?(bool=True)
UpdateLlmServerRequest: name?, backend_type?, base_url?, api_key?, is_active?  (all optional)
EnabledModelsRequest: enabled_models(list[str])
AvailableModelsResponse: models(list[{id: str}])
EnabledModelInfo: server_id(str), server_name(str), model_id(str)
EnabledModelsListResponse: models(list[EnabledModelInfo])
```

### Probe logic

Uses `PythonLLMClient` (already a dependency):
- Instantiate `LlamaSwapAPIClient` or `OpenAIAPIClient` based on `backend_type`
- Call `client.list_models()` to get available model IDs
- Resolve `$ENV_VAR` api keys before passing to client
- Return 502 on connection failure

### Page Component

**File**: `frontend/src/admin/pages/LlmServersPage.tsx`
**Route**: `/admin/llm-servers`
**UI Library**: Mantine (not Ant Design — adapt reference accordingly)

#### Main view
- Page title "LLM Servers" + "Add Server" button
- Table with columns: Name, Type (badge), Base URL, Key (Yes/No), Models (count), Active (badge), Actions
- Actions per row: **Models** button, **Edit** button, **Delete** button (with confirm)

#### Add/Edit modal
- Fields: Name (required), Backend Type (select: llama-swap / openai), Base URL (required), API Key (password input, placeholder "(unchanged if empty)" on edit), Active (switch)
- On edit: also show "Select Models" button that opens the models modal

#### Models modal
- Title: "Models — {server name}"
- Auto-probes server on open
- Filter input at top
- Checkbox list of all models (union of probed + already enabled), scrollable
- Shows spinner while probing
- Save button persists selected models
