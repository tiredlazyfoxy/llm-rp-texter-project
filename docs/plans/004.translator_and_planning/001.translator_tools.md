# Feature 004 Step 003: Translate-to-English Button

## Context

Users write in non-English languages and need a quick way to translate their input to English before sending. This feature adds a translate icon button next to all text inputs (user RP chat, admin LLM chat, document editors). The original text is preserved in browser state with a revert option — revert disappears if the user edits the translated text.

Separate backend endpoints for admin and user chat (different auth roles).

## Backend

### 1. Pydantic Schemas

**File:** `backend/app/models/schemas/llm_chat.py` — add:

```python
class TranslateRequest(BaseModel):
    text: str
    model_id: str

class TranslateResponse(BaseModel):
    translated_text: str
```

### 2. Service Function

**File:** `backend/app/services/llm_chat.py` — add `translate_to_english()`:

- Gets client via existing `get_llm_client_for_model(model_id)`
- Non-streaming `client.chat()` call with temperature 0.1
- System prompt: "You are a translator. Translate the user's message into English. Output ONLY the translated text, nothing else. If the text is already in English, return it unchanged."
- Returns stripped response string

### 3. Admin Endpoint

**File:** `backend/app/routes/llm_chat.py` — add `POST /api/llm/translate`:

- Requires `editor` role (reuse existing `_require_editor`)
- Accepts `TranslateRequest`, returns `TranslateResponse`
- Calls `llm_chat_service.translate_to_english(req.text, req.model_id)`

### 4. User Chat Endpoint

**File:** `backend/app/routes/chat.py` — add `POST /api/chats/translate`:

- Requires `player` role (reuse existing `_require_player`)
- Accepts `TranslateRequest`, returns `TranslateResponse`
- Same service call

### 7. Shared Hook: `useTranslation`

**New file:** `frontend/src/hooks/useTranslation.ts`

Interface:
```typescript
interface UseTranslationOptions {
  getValue: () => string;
  setValue: (text: string) => void;
  getModelId: () => string | null;
  translateFn: (req: TranslateRequest) => Promise<TranslateResponse>;
}

// Returns:
{
  isTranslating: boolean;   // loading state
  canRevert: boolean;       // original stored AND text not edited since translation
  handleTranslate: () => Promise<void>;
  handleRevert: () => void;
  onInputChange: (newValue: string) => void;  // must be called on every input change
}
```

Logic:
- On translate: store original text + translated text in refs, call API, set value to result
- `canRevert` = original exists AND current value === translated text (exact match)
- On revert: restore original, clear refs
- `onInputChange`: if value differs from translated text → discard original (canRevert becomes false)
