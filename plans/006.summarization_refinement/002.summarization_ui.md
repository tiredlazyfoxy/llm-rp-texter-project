# Feature 006 Step 002: Summarization Refinement — Frontend

## Context

Backend from Feature 006 Step 001 provides: SSE-streamed compact endpoint (two-phase: memory extraction + summarization with token streaming), un-summarize DELETE endpoint, variant-aware compaction. This step wires all of it into the frontend.

## Requirements (frontend portion)

1. **SSE compact client** — consume streaming compact endpoint (phase, token, tool_call, compact_done events)
2. **Streaming summary preview** — show summary text being written token-by-token during compaction
3. **Debug visibility** — show `add_memory` tool calls and phase labels in debug mode
4. **Allow compacting last message** — remove `turn_number < currentTurn` restriction
5. **Variant-aware compact** — pass selected variant index when compacting current turn
6. **Un-summarize UI** — undo button on last summary, surgical MobX update
7. **Surgical MobX update** — no full chat reload after compaction
8. **Expand/collapse intact** — lazy-load on expand, destroy on collapse (no changes needed)

---

## Implementation Steps

### 1. SSE Compact Client

**Files:**
- `frontend/src/api/chat.ts` — replace `compactChat` REST call with SSE streaming
- `frontend/src/types/chat.d.ts` — add `CompactRequest.variant_index`

**Details:**
- New function: `compactChatStream(chatId, req, handlers)` using `_streamChat` pattern
- Handlers interface:
  ```typescript
  interface CompactSSEHandlers {
    onPhase?: (phase: string) => void;
    onToken?: (content: string) => void;
    onToolCallStart?: (toolName: string, args: Record<string, string>) => void;
    onToolCallResult?: (toolName: string, result: string) => void;
    onDone?: (response: CompactResponse) => void;
    onError?: (detail: string) => void;
  }
  ```
- Reuse `_streamChat` with a `compact_done` event case (maps to `onDone`)
- Update `CompactRequest` type: add `variant_index?: number`

### 2. ChatStore Compact with SSE + Debug State

**Files:**
- `frontend/src/user/stores/ChatStore.ts` — update `compactUpTo()`, add compact debug observables

**Details:**
- New observables:
  - `compactPhase: string | null = null`
  - `compactToolCalls: Array<{ tool_name: string; arguments: Record<string, string>; result?: string }> = []`
  - `compactStreamingContent: string = ""` — summary text being streamed
- `compactUpTo(messageId, variantIndex?)`:
  - Reset compact debug state
  - Set `isCompacting = true`
  - Call `compactChatStream()` with handlers:
    - `onPhase`: update `compactPhase`
    - `onToken`: append to `compactStreamingContent`
    - `onToolCallStart`: push to `compactToolCalls`
    - `onToolCallResult`: update matching entry
    - `onDone`: push final summary to `summaries`, filter messages (existing logic), reset compact state
    - `onError`: set error, reset compact state
  - After `onDone`: reload memories (since `add_memory` was called)

### 3. Allow Compacting Last Message + Variant Passing

**Files:**
- `frontend/src/user/components/MessageHistory.tsx` — change `showCompact` condition

**Details:**
- Change `msg.turn_number < currentTurn` to `msg.turn_number <= currentTurn`
- When compacting the current turn's message, pass `chatStore.viewingVariantIndex` as `variant_index`

### 4. Compact Progress UI with Streaming Summary

**Files:**
- `frontend/src/user/components/MessageHistory.tsx` or new `CompactProgress.tsx` component

**Details:**
- When `chatStore.isCompacting`, show a progress card at the bottom of the message list:
  - **Always visible:** Phase label ("Extracting memories..." / "Writing summary...")
  - **Summary streaming:** When `compactPhase === "summarization"` and `compactStreamingContent` is not empty, render the streaming summary text with `ReactMarkdown` (same styling as `SummaryBlock` but with a subtle "writing" indicator)
  - **Debug mode additions:** Show `compactToolCalls` using existing `ToolCallTrace` component pattern — `add_memory` calls visible with arguments and results
  - **Non-debug mode:** Simple spinner + phase text + streaming summary text
- Once `compact_done` arrives, the streaming preview is replaced by the actual `SummaryBlock`

### 5. Un-summarize UI

**Files:**
- `frontend/src/api/chat.ts` — add `unsummarizeLast()`
- `frontend/src/user/stores/ChatStore.ts` — add `unsummarizeLast()` action
- `frontend/src/user/components/SummaryBlock.tsx` — add undo button on last summary

**Details:**
- API: `DELETE /api/chats/${chatId}/summaries/${summaryId}` → returns `ChatMessage[]` (restored messages)
- Store action `unsummarizeLast(summaryId)`:
  - Call API
  - Remove summary from `summaries` array
  - Insert restored messages into `currentChat.messages` at correct position (sorted by `turn_number`)
  - Clear expanded messages for this summary if any
  - Reload memories (cleanup if needed)
- UI: `SummaryBlock` gets `isLast: boolean` prop
  - When `isLast`, show "Undo summary" button (e.g., `IconArrowBackUp` from Tabler)
  - Confirmation dialog before executing
  - Disabled when `isCompacting` or `isSending`
- `MessageHistory` passes `isLast` based on summary index in `chatStore.summaries`

## Verification

1. **Un-summarize**: Compact some messages, click undo on the last summary → messages reappear in chat, summary disappears, no full page reload
2. **Compact last message**: On the last assistant message, click fold → summary created including current turn
3. **Variant-aware compact**: Generate multiple variants, select one via variant navigator, compact → summary reflects the selected variant's content
4. **Streaming summary**: During compaction, see the summary text being written token-by-token → when done, it becomes a proper SummaryBlock
5. **Memory extraction (debug)**: Compact with debug mode on → see "Memory Extraction" phase, `add_memory` tool calls visible, then "Summarization" phase with streaming text
6. **Non-debug mode**: Compact without debug → see phase status text + streaming summary, no tool call details
7. **Expand/collapse**: After compaction, expand summary → original messages appear at 60% opacity. Collapse → messages disappear, DOM cleaned up
8. **Partial update**: After compaction, verify only the affected messages are removed and summary is added — no full chat reload, no scroll jump
