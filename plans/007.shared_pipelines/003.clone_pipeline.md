# Feature 007 Step 003 ‚Äî Clone Pipeline (shadow-then-save)

Goal: add a **Clone** button to `PipelineEditPage` that opens a "shadow" edit page ‚Äî pre-filled from the source pipeline with `name = "<source name> (clone)"` ‚Äî that exists only in the browser. Pressing **Save** materializes the record via `POST /api/admin/pipelines` and redirects to the real edit page; pressing **Back** discards it.

Depends on Step 002 (pipeline admin UI + per-stage routes).

## 0. Scope notes

- **Frontend-only.** No backend changes. The existing `POST /api/admin/pipelines` endpoint already accepts `name`, `description`, `kind`, `system_prompt`, `simple_tools`, `pipeline_config`, `agent_config` via `CreatePipelineRequest` (see `backend/app/models/schemas/pipeline.py` lines 58‚Äì65). Step 001 wired its service + route. Verified: no schema, validator, or DB change required.
- **Out of scope** (mention only ‚Äî do NOT implement here): replacing the `PipelinesListPage` "Create" modal with a `cloneFrom`-less variant of the same shadow page. That's a possible follow-up; this step keeps the modal as-is and adds Clone next to it.
- Backend currently does NOT enforce unique `pipeline.name` (`backend/app/db/pipelines.py::create` and `backend/app/services/pipelines.py::create_pipeline` perform no name lookup). Cloning the same source twice will produce two `"X (clone)"` rows. Matches `clone_world` semantics (`world_editor.clone_world` also doesn't dedupe). Decision: **leave as-is**, do not add uniqueness ‚Äî uniqueness is a separate concern.

## 1. Routing changes

**Edit** `frontend/src/admin/App.tsx` ‚Äî `AdminContent()`. The current matcher block (line ~38‚Äì42):

```tsx
if (path.startsWith("/admin/pipelines")) {
  if (/\/admin\/pipelines\/\d+\/stage\/\d+/.test(path)) return <PipelineStageEditPage />;
  if (/\/admin\/pipelines\/\d+/.test(path)) return <PipelineEditPage />;
  return <PipelinesListPage />;
}
```

becomes:

```tsx
if (path.startsWith("/admin/pipelines")) {
  if (/\/admin\/pipelines\/\d+\/stage\/\d+/.test(path)) return <PipelineStageEditPage />;
  if (path === "/admin/pipelines/new" || path.startsWith("/admin/pipelines/new?")) return <PipelineEditPage />;
  if (/\/admin\/pipelines\/\d+/.test(path)) return <PipelineEditPage />;
  return <PipelinesListPage />;
}
```

The `new` matcher must come **before** the `\d+` matcher. `\d+` would not match the literal `new` anyway, but ordering this explicitly leaves zero ambiguity for the next reader.

URL convention: `/admin/pipelines/new?cloneFrom=<sourceId>`. Justification: keeps `PipelineEditPage` as the single owner of pipeline-edit UI; the `new` segment makes "no id yet" obvious in URL bars and history; `cloneFrom` is a query param so it's optional (future "create blank" use case at the same URL works without rework).

## 3. PipelineEditPage state changes

The page now has two modes. Keep one component, branch internally.

### 3.1 URL parsing

Replace `extractPipelineId()` (lines 50‚Äì53) with a `parseRoute()` helper:

```tsx
type PageMode = "edit" | "shadow";
interface RouteInfo { mode: PageMode; pipelineId: string | null; cloneFromId: string | null; }

function parseRoute(): RouteInfo {
  const path = window.location.pathname;
  if (path === "/admin/pipelines/new") {
    const cloneFrom = new URLSearchParams(window.location.search).get("cloneFrom");
    return { mode: "shadow", pipelineId: null, cloneFromId: cloneFrom };
  }
  const m = path.match(/\/admin\/pipelines\/(\d+)$/);
  return { mode: "edit", pipelineId: m ? m[1] : null, cloneFromId: null };
}
```

### 3.2 Component state

Replace `const pipelineId = extractPipelineId();` (line 60) with:

```tsx

const { mode, pipelineId, cloneFromId } = route;
```

The existing `pipeline: PipelineItem | null` state remains. In shadow mode it stays `null` for the lifetime of the shadow ‚Äî the header title falls back to `"New pipeline (clone)"` (see ¬ß3.4).

### 3.3 Loader (`load`)

Current loader (lines 82‚Äì105) fetches `getPipeline(pipelineId)`. Branch on `mode`:

```tsx
const load = useCallback(async () => {
  setLoading(true);
  setError(null);
  try {
    if (mode === "shadow") {
      // Blank-shadow path is out of scope; today we always have cloneFromId.
      if (!cloneFromId) {
        setError("Missing cloneFrom parameter");
        return;
      }
      const src = await getPipeline(cloneFromId);
      // pipeline stays null ‚Äî this is a shadow, no PipelineItem yet.
      setName(`${src.name} (clone)`);
      setDescription(src.description);
      setKind(src.kind);
      setSystemPrompt(src.system_prompt);
      try { setSimpleTools(JSON.parse(src.simple_tools || "[]")); } catch { setSimpleTools([]); }
      try {
        const parsed = JSON.parse(src.pipeline_config || "{}");
        setPipelineConfig({ stages: parsed.stages || [] });
      } catch {
        setPipelineConfig({ stages: [] });
      }
      // agent_config is carried over via a ref so handleSave can ship it verbatim.
      shadowAgentConfigRef.current = src.agent_config ?? "{}";
      return;
    }
    if (!pipelineId) return;
    const p = await getPipeline(pipelineId);
    setPipeline(p);
    setName(p.name);
    setDescription(p.description);
    setKind(p.kind);
    setSystemPrompt(p.system_prompt);
    try { setSimpleTools(JSON.parse(p.simple_tools || "[]")); } catch { setSimpleTools([]); }
    try {
      const parsed = JSON.parse(p.pipeline_config || "{}");
      setPipelineConfig({ stages: parsed.stages || [] });
    } catch {
      setPipelineConfig({ stages: [] });
    }
  } catch (e) {
    setError(e instanceof Error ? e.message : "Failed to load pipeline");
  } finally {
    setLoading(false);
  }
}, [mode, pipelineId, cloneFromId]);
```

`shadowAgentConfigRef` is declared at the top of the component:

```tsx
const shadowAgentConfigRef = useRef<string>("{}");
```

Reason for a ref (not state): `agent_config` is currently not edited anywhere in this page; it's just a passthrough to preserve fidelity from source to clone. Holding it in a ref avoids spurious re-renders.

## 4. Save dispatch (`handleSave`)

Replace the body of `handleSave` (lines 113‚Äì136) with:

```tsx
const handleSave = async () => {
  setSaving(true);
  setError(null);
  setSuccess(null);
  try {
    if (mode === "shadow") {
      const created = await createPipeline({
        name,
        description,
        kind,
        system_prompt: systemPrompt,
        simple_tools: JSON.stringify(simpleTools),
        pipeline_config: JSON.stringify(pipelineConfig),
        agent_config: shadowAgentConfigRef.current,
      });
      window.location.href = `/admin/pipelines/${created.id}`;
      return;
    }
    if (!pipelineId) return;
    const req: UpdatePipelineRequest = {
      name,
      description,
      kind,
      system_prompt: systemPrompt,
      simple_tools: JSON.stringify(simpleTools),
      pipeline_config: JSON.stringify(pipelineConfig),
    };
    const updated = await updatePipeline(pipelineId, req);
    setPipeline(updated);
    setSuccess("Pipeline saved");
    setTimeout(() => setSuccess(null), 3000);
  } catch (e) {
    setError(e instanceof Error ? e.message : "Failed to save pipeline");
  } finally {
    setSaving(false);
  }
};
```

Add `createPipeline` to the import from `../../api/pipelines` (currently lines 39‚Äì44 only import `deletePipeline`, `getPipeline`, `getPipelineConfigOptions`, `updatePipeline`).

After the redirect on shadow-save, `setSaving(false)` is reached (the `finally` runs before navigation completes) ‚Äî harmless because the page is unmounting; not worth special-casing.

The chain-mode stage list renders an `IconSparkles` ActionIcon (lines 450‚Äì459) that navigates to `/admin/pipelines/${pipelineId}/stage/${idx}`. In shadow mode there is no `pipelineId` to put in the URL. Two acceptable UX options:

**Picked: disable + tooltip.** Leave the icon visible but disabled, with a `title` explaining why. Concrete change at lines 450‚Äì459:

```tsx
<ActionIcon
  variant="subtle"
  size="sm"
  title={mode === "shadow" ? "Save the clone first to edit stage prompts" : "Edit Prompt"}
  disabled={mode === "shadow"}
```

## 8. Edge cases / decisions

1. **Double-click Clone**: navigation happens via `window.location.href` so the second click is racing the unload ‚Äî harmless.
2. **`cloneFrom` points to a deleted pipeline**: `getPipeline(cloneFromId)` returns 404; the existing `catch` sets `error` and the page shows "Failed to load pipeline" ‚Äî acceptable.
3. **`cloneFrom` is non-numeric**: query param is taken verbatim; `getPipeline` will reject. No client-side regex needed ‚Äî the backend is the source of truth.
4. **Source has `agent_config` populated**: `agent_config` is preserved via the ref and posted on save. Even though the page never edits it, this guarantees a clone of an `agentic` pipeline (when that mode lands) keeps its config.
5. **Source `kind = "agentic"`**: the `Kind` `Select` has `agentic` disabled (line 221), but the Select can still *display* the value for an agentic source ‚Äî Mantine renders disabled options when they match the current value. The clone will save with `kind="agentic"`. Acceptable.
6. **Name collision**: documented in ¬ß0; no change.
7. **User edits Name to empty before Save**: backend `CreatePipelineRequest.name: str` has no `min_length`; an empty string is accepted (matches existing `PipelinesListPage` modal which only enforces non-empty client-side). Mirror that ‚Äî make Save reject empty `name` client-side with the same message: `if (!name.trim()) { setError("Name is required"); return; }`. Add this guard at the top of `handleSave` for both modes (no behavior change in edit mode where the field starts populated, but makes shadow safe).

## 9. Acceptance criteria

- A Clone button appears on `/admin/pipelines/<id>` for both editor and admin roles, next to Save.
- Clicking Clone navigates to `/admin/pipelines/new?cloneFrom=<id>`.
- The page at `/admin/pipelines/new?cloneFrom=<id>` loads with form fields populated from the source pipeline; `name` is `"<source.name> (clone)"`; the title reads `"<source.name> (clone)"`.
- Delete button is hidden in shadow mode. Clone button is hidden in shadow mode.
- The `IconSparkles` per-stage edit link is disabled in shadow mode with the tooltip "Save the clone first to edit stage prompts".
- Pressing Save in shadow mode posts to `POST /api/admin/pipelines` and the page redirects to `/admin/pipelines/<new-id>`. Reloading that URL shows the saved clone.
- Pressing Back in shadow mode returns to `/admin/pipelines`; the source pipeline is unchanged; no new row was created.
- Source pipeline is unchanged after a successful clone (sanity check: same `modified_at`, same fields).
- `kind="chain"` clone preserves the full `pipeline_config.stages` array byte-for-byte (round-trip via `JSON.parse`/`JSON.stringify`).
- `kind="simple"` clone preserves `system_prompt` and `simple_tools` byte-for-byte.
- `agent_config` is carried over verbatim from source to clone.
- `npx tsc --noEmit` exits 0; `npx vite build` succeeds.
- Routing: `/admin/pipelines/new` resolves to `PipelineEditPage`; `/admin/pipelines/<id>` still resolves to `PipelineEditPage`; `/admin/pipelines/<id>/stage/<idx>` still resolves to `PipelineStageEditPage`.
