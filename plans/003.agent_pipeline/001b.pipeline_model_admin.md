# Feature 003 Step 001 ‚Äî Pipeline Config Model + Admin UI

## Context

The pipeline is a **chain approach**: a planning step (LLM with tools ‚Üí structured JSON) followed by a writing step (LLM ‚Üí prose). The structure is extensible ‚Äî currently 2 step types but designed to support more later.

Each step's system prompt has two parts: a **coded part** (hardcoded in Python ‚Äî data structures, I/O formats, tool instructions) and an **admin-editable part** (free text stored per stage in the pipeline config). The admin part lets world creators add instructions about world logic, tone, and intentions. These prompts are editable via the LLM chat assistant (same pattern as `WorldFieldEditPage`).

A world-level `generation_mode` field controls which flow is active: `"simple"` (current single-step), `"chain"` (pipeline), or `"agentic"` (future sub-agent approach, disabled for now). Two separate JSON config fields (`pipeline` and `agent_config`) store settings for each mode independently ‚Äî switching modes preserves both configs.

This step defines the config schema, creates prompt skeletons, adds hidden stat support, and builds admin UI. No pipeline execution ‚Äî that's step 2.

## 2. Pipeline Config Schema

### 2a. Pydantic Models

**New file**: `backend/app/models/schemas/pipeline.py`

```python
class PipelineStage(BaseModel):
    step_type: str              # "planning" | "writing" (extensible later)
    prompt: str = ""            # admin free-text, editable via LLM chat
    max_agent_steps: int | None = None  # only for tool-calling types (planning)

class PipelineConfig(BaseModel):
    stages: list[PipelineStage] = []

class StatUpdateEntry(BaseModel):
    name: str
    value: str

class GenerationPlanOutput(BaseModel):
    collected_data: str = ""
    stat_updates: list[StatUpdateEntry] = []
    decisions: list[str] = []
```

### 2b. Design Notes

- `PipelineConfig` has no `enabled` flag ‚Äî enablement is controlled by `World.generation_mode == "chain"`
- **Step types** (predefined, validated on save):
  - `"planning"` ‚Äî tool-calling agent, produces structured JSON (collected_data, decisions, stat_updates). `max_agent_steps` applies.
  - `"writing"` ‚Äî prose generation from plan. `max_agent_steps` ignored.
- **Default pipeline** when admin switches to chain mode:
  ```json
  {"stages": [
    {"step_type": "planning", "prompt": "", "max_agent_steps": 10},
    {"step_type": "writing", "prompt": ""}
  ]}
  ```
- Migration: `PipelineConfig.model_validate_json("{}")` ‚Üí empty stages list. No DB migration needed.
- Model selection, temperature ‚Äî stay as world-level / session-level settings, **not** in pipeline config.
