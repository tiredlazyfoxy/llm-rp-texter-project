## 5. Pipeline Service

### 5a. New File: `backend/app/services/pipeline_service.py`

```python
async def generate_pipeline_response(
    session_id: int,
    user_id: int,
    user_message: str,
    caller_role: str,
) -> AsyncGenerator[str, None]:
```

**Flow:**

1. Load session, world, resolve pipeline config
2. Save user message (same as single-step)
3. **Thinking phase**:
   - Yield `phase("thinking")`
   - Build context via `build_chat_context()`
   - Get tools from `get_chat_tools(world_id, session_id)`
   - Wrap tools with SSE emission (following `_run_with_tools` pattern from `llm_chat.py`)
   - Get thinking LLM client via `get_llm_client_for_model(thinking_model_id)`
   - Call `client.chat_with_tools(messages, tools_definitions, tools, system, options, max_loops, stream, on_delta)`
   - Yield `status`, `tool_call_start/result`, `thinking` events
   - Parse final response as JSON → `GenerationPlanOutput`
   - JSON fallback: `json.loads()` → regex `\{[\s\S]*\}` → error event
   - Validate + apply stat updates via `validate_and_apply_stat_updates()`
   - Yield `stat_update` (always, even if empty — marks phase boundary)
4. **Writing phase**:
   - Yield `phase("writing")`, `status("Writing...")`
   - Build writer context: summaries + clean messages (user/assistant only, no tool_calls) + plan message
   - Get writing LLM client
   - Stream via `client.chat()`, yield `token` events
5. **Finalize**:
   - Save assistant message with `content` = prose, `generation_plan` = JSON string
   - Increment turn, save snapshot
   - Yield `done`

### 5b. Visibility Rules

Based on `caller_role`:
- Regular users: skip `thinking`, `tool_call_start/result` events. Only emit `status` events.
- Editor+: emit all events.
- `phase` always emitted.

## 10. Verification

1. **Single-step** (pipeline disabled): rich prompt includes location/NPCs/rules/stats. Stat updates parsed and validated.
2. **Pipeline** (pipeline enabled): thinking agent calls tools, produces valid JSON plan. Stats validated and applied. Writer streams prose.
3. **SSE events**: `phase(thinking)` → `status` → tool events → `stat_update` → `phase(writing)` → tokens → `done`
4. **Visibility**: regular user sees only `status` + `phase` events. Editor+ sees all events.
5. **Per-session overrides**: editor changes thinking/writing models → takes effect
6. **Temperature**: regular user adjusts per-stage temperatures → applies
7. **Regeneration**: both stages re-run, old variant deactivated, stats reverted from pre-turn snapshot
8. **JSON failure**: thinking agent produces bad JSON → error SSE event, no partial message saved
9. **Tool errors**: individual tool errors returned to agent, agent continues
10. **Import/export**: new columns handled correctly
